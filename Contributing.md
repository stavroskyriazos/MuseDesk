---

## 🧠 Contribution Philosophy

* Work collaboratively and communicate regularly
* Keep code clean, simple, and well-structured
* Ensure all changes are tested before merging
* Maintain clear documentation for all features

---

## 🔧 How to Contribute

### Create a Branch

Each feature or fix should be developed in a separate branch:

```bash
git checkout -b feature-name
```

---

### Make Changes

* Write clear and readable code
* Follow the existing project structure
* Test your changes before committing

---

### Commit Changes

Use meaningful commit messages:

```bash
git commit -m "Added login API functionality"
```

---

### Push to Repository

```bash
git push origin feature-name
```

---

### Create Pull Request

* Open a pull request to the `main` branch
* Describe your changes clearly
* Wait for review before merging

---

## 🧪 Testing

* Test all features before submitting changes
* Ensure no existing functionality is broken

---

## 📂 Code Structure Guidelines

* Backend code → `/backend`
* Frontend code → `/frontend`
* Documentation → `/docs`

---

## 📝 Reporting Issues

* Use GitHub Issues to report bugs or suggest features
* Provide clear descriptions and steps to reproduce
